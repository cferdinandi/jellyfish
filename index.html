Getting Started

1. Include Jellyfish on your site.

<script src="jellyfish.js"></script>

Don't forget to add the img/loading.gif image to your site, too. If you change its location, update the loadingSrc variable in jellyfish.js

Heads Up! Make sure to put your JS file in the footer or Jellyfish won't work (it's better for performance, too).

2. Add the markup to your HTML.

You can turn any a, span, or div tag into an image lazy loader by adding the [data-lazy-load] data attribute. Include a [data-img] attribute so that Jellyfish knows what image to serve when loading.

<span data-lazy-load data-img="img/elephant.jpg"></span>

Fallback Links

If users should have access to the image even if Jellyfish isn't compatible with their browser or the JavaScript file fails to load, use a link that points to the image source instead of a span or div.

<a data-lazy-load data-img="img/friends.jpg" href="img/friends.jpg">
	View Photo
</a>

Adding Image Attributes

You can add titles, classes and more using data attributes.

  • [data-class] - Add one or more clases to the image.
  • [data-height] - Add a height value to the image.
  • [data-width] - Add a width value to the image.
  • [data-title] - Add a title attribute to the image.
  • [data-loading] - Replace the default loading image with your own.
<a
	data-lazy-load data-img="img/ice-skaters.jpg"
	data-class="img-border"
	data-height="791"
	data-width="1024"
	data-title="Old-timey ice skaters"
	data-loading="img/loading.gif"
	href="img/ice-skaters.jpg"
>
	View Photo
</a>

3. Pat yourself on the back.

Seriously, you're done. Nice work! Modify as needed.

Customizing Script Variables

There are two global settings you can adjust in the jellyfish.js file itself to change the Jellyfish defaults.

  • loadingIcon - Specifies the location of the loading icon file.
  • offset - Determines how far below the fold (in pixels) to begin loading images. It's set to 0 by default, which means images load as they enter the viewport.

Browser Compatability

Jellyfish works with all modern browsers, and IE 9 and above.

Jellyfish is built with modern JavaScript APIs, and uses progressive enhancement. If the JavaScript file fails to load, or if your site is viewed on older and less capable browsers, links to the image files will be displayed instead (if you've used that method).
